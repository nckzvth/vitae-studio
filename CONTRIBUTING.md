# Contributing

Thank you for helping make Vitae Studio safer and more useful.

1. Create a focused branch from `main`.
2. Keep all example CV data fictional.
3. Add or update tests and documentation with behavior changes.
4. Run `npm run validate` before opening a pull request.
5. Describe privacy, accessibility, pagination, or deployment implications in the pull request.

Please use conventional, imperative commit subjects. Security or privacy-sensitive reports should not include real CV data.
